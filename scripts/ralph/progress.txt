## Codebase Patterns
- Next.js app uses port 7000 for dev server (`npm run dev`)
- Project root is `/Users/gloucore/dev/claude-ralph-loop/`
- Tailwind CSS v4 with @tailwindcss/postcss setup
- Tailwind v4: Custom colors via `@theme inline` in globals.css (e.g., `--color-brand-coral: #FF7593`)
- Use Tailwind classes like `bg-brand-coral`, `text-brand-green`, `bg-dark-alt`
- Prisma 7 uses `prisma.config.ts` for DATABASE_URL (not in schema.prisma)
- PrismaClient singleton exported from `src/lib/db.ts` as `db`
- Resend client in `src/lib/email.ts`, templates in `src/lib/email-templates.ts`
- Auth utilities in `src/lib/auth.ts`: hashPassword, verifyPassword, createSession, getSession, destroySession
- Use `getSession()` in server components/actions to get current user (returns session.user or null)
- After schema changes, always run `npx prisma generate` before building
- `useSearchParams()` must be wrapped in `<Suspense>` for Next.js 16+ static builds
- **Server component pages with Prisma:** Use dynamic import inside async function to avoid build-time init issues: `const { getSession } = await import("@/lib/auth")`
