## Codebase Patterns
- Use `vitest` for unit tests with jsdom environment
- Path alias `@/*` resolves to `./src/*` in both tsconfig and vitest.config
- Test setup file at `src/__tests__/setup.ts` imports @testing-library/jest-dom
- Run `npm run test -- --run` for single pass, `npm run test` for watch mode
- Prisma 7: `prisma db push` has no --skip-generate flag (use plain `prisma db push`)
- Auth utility tests go in `src/lib/__tests__/auth.test.ts`, only pure functions (no db/cookie access)
- Vitest must exclude `**/e2e/**` to avoid parsing Playwright tests (different test runner)
- Playwright webServer auto-starts dev server; use reuseExistingServer: !isCI for local dev
