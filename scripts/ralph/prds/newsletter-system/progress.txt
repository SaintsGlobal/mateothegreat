# Newsletter System Progress Log

## Codebase Patterns
- Project uses ES modules (`"type": "module"` in package.json)
- Zod v4 recursive schemas: define TypeScript types manually first, then use `z.ZodType<T>` annotation for recursive schema variables
- For nested block structures (expandable, tabs), limit to base blocks to avoid circular type issues
- Project root is /Users/gloucore/dev/claude-ralph-loop (not the prd directory)
- Dev server runs on port 7000
- Template UUIDs must be valid (not all-zeros) - use format like `a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d`
- Test files go in `src/lib/__tests__/` directory
- Vitest config at project root: `vitest.config.ts`
- Tailwind v4: use `@import "tailwindcss"` in globals.css (not @tailwind directives)
- Block components: `src/components/newsletter/blocks/{type}-block.tsx` with `{Type}BlockComponent` naming
- Client components using react-markdown need "use client" directive
- Prisma 7: config in `prisma.config.ts` at project root, schema in `prisma/schema.prisma`, use dotenv for DATABASE_URL
