## Codebase Patterns
- Use Server Actions for form submissions (not API routes when possible)
- Prisma client imported from src/lib/db.ts
- Dark theme with brand colors: cyan (#4CC2D5), coral (#FF7593), gold (#FFC927)
- Components use Tailwind CSS classes
- Protected routes check session and redirect to /signin
- PB-001 billing models were merged to main, so ralph/profile-billing was created from main
- Use @@index on Prisma foreign key fields for query performance
- Run `npx prisma generate` after schema changes to update TypeScript types
- Reuse existing Server Actions (e.g., updateProfile in auth.ts) rather than creating new ones
